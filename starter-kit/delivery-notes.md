# Devops Delivery Notes

## Overview
This project was implemented using DevOps principles to ensure efficient development, collaboration, and continuous improvement. The workflow demonstrates the three core DevOps principles: Flow, Feedback, and Learning.

### 1. Flow
Flow focuses on how smoothly work moves from the moment the idea is created until it running in production.

In this project:
- Work was developed in a **feature branch (feature/starter-kit-files)** instead of directly on the main branch. 
- Changes were committed in small, manageable increments.
- A structured git workflow was followed:
    *feature -> develop -> main*  
    This ensured that work moved in a controlled and organized manner, reducing errors and improving efficiency.

### 2. Feedback
Feedback ensures that the delivery system detects quickly detects when something goes wrong

In this project:
- A Pull Request (PR) was created from the feature branch to the develop branch
- The PR allowed for review of changes before merging
- This process helps catch errors, improve quality, and validate decisions before integration. 

### 3. Learning
Learning ensures every incident, failure or mistake shoold lead to an improvement in the system. 

In this project:
- Challenges such as Git branch management and repository synchronization were encountered and resolved.
- The team improved understanding of:
  - Git workflows
  - Branching strategies
  - Cloud architecture design principles
- Lessons learned were applied to improve the structure and clarity of the project.


## Conclusion

The workflow demonstrates the three DevOps principles:
- **Flow** through structured branching and commits
- **Feedback** through Pull Requests and review
- **Learning** through problem-solving

These principles ensure efficient, reliable, and continuously improving software delivery.
